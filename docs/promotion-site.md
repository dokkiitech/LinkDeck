# LinksDeck - プロモーションサイト用ドキュメント

## アプリ概要

### キャッチコピー
**URLを見返さない生活もうやめましょう**

### LinksDeck（リンクスデッキ）とは？

LinksDeckは、Webブラウジング中に見つけた有益な情報や後で読みたい記事のURLを簡単に保存・整理できるモバイルアプリケーションです。シンプルで直感的なUIと、AIによる要約機能（準備中）により、効率的に情報を管理できます。

---

## コアコンセプト

### 誰のためのアプリか？

- **情報収集が多い方**: 日々多くの記事やWebページを読む方
- **後で読むリストが溜まってしまう方**: ブックマークやリーディングリストが整理できない方
- **効率的に情報を整理したい方**: タグで分類して、必要な時にすぐ見つけたい方
- **複数デバイスで同期したい方**: スマホでもタブレットでも同じデータを扱いたい方

### どんな課題を解決するか？

1. **ブックマークの整理が大変** → タグで簡単に分類・検索
2. **後で読もうと思って忘れる** → アーカイブ機能で未読/既読を管理
3. **URLの内容を覚えていない** → タイトルとURLを自動取得
4. **他のアプリから保存が面倒** → iOSショートカットで簡単共有（将来的にはShare Extension対応予定）
5. **長い記事を読む時間がない** → AI要約機能で素早く内容把握（準備中）

## 主要機能

### ✅ 実装済み機能

#### 1. **リンク管理**
- URLとタイトルの保存
- リンク一覧表示（カード型UI）
- リンク詳細表示
- リンクの削除
- アーカイブ機能（未読/既読の管理）

#### 2. **タグ管理**
- 自由にタグを作成・削除
- リンクに複数タグを紐付け
- タグによるフィルタリング
- タグ一覧表示

#### 3. **認証・セキュリティ**
- メール/パスワード認証
- ゲストログイン機能
- ユーザーごとにデータを完全分離
- Firebase Authenticationによる安全な認証

#### 4. **URL共有機能**
- iOSショートカット経由での共有
- Safari、Chrome、その他のアプリから保存可能
- URLとページタイトルを自動取得
- ワンタップで保存完了

#### 5. **設定・カスタマイズ**
- アカウント情報表示
- Gemini APIキー設定（UI実装済み）
- ログアウト機能

### 🚧 準備中の機能

#### 1. **iOS Share Extension（完全なネイティブ共有機能）**
- 標準の共有メニューからLinksDeckを直接選択
- より自然で快適な共有体験
- App Groupsによるメインアプリとの連携

#### 2. **AI要約機能（Gemini API統合）**
- Gemini APIによる記事の自動要約
- 長い記事も数行で内容を把握
- 要約のキャッシュで高速表示
- ユーザー独自のAPIキーで利用

#### 3. **UI/UX改善**
- ダークモード対応
- タイムライン表示
- 高度な検索機能
- 無限スクロール

## データモデル

### ユーザーデータ
```typescript
{
  uid: string;              // ユーザーID
  email: string | null;     // メールアドレス
  displayName: string | null; // 表示名
  createdAt: Timestamp;     // 作成日時
  geminiApiKey?: string;    // Gemini APIキー（暗号化済み）
}
```

### リンクデータ
```typescript
{
  id: string;               // リンクID
  userId: string;           // ユーザーID
  url: string;              // URL
  title: string;            // タイトル
  tags: string[];           // タグ名の配列
  isArchived: boolean;      // アーカイブ状態
  createdAt: Timestamp;     // 作成日時
}
```

### タグデータ
```typescript
{
  id: string;               // タグID
  userId: string;           // ユーザーID
  name: string;             // タグ名
  createdAt: Timestamp;     // 作成日時
}
```

## ユースケース

### ケース1: ビジネスパーソン
**シーン**: 仕事中に見つけた業界ニュースや参考記事を保存
- **Before**: ブラウザのブックマークが散らかって見返せない
- **After**: LinksDeckでタグ管理し、プロジェクトごとに整理

### ケース2: 学生・研究者
**シーン**: レポートや論文執筆のための参考資料管理
- **Before**: 複数のブラウザタブを開きっぱなしで混乱
- **After**: タグで分類して、テーマごとにすぐアクセス

### ケース3: 趣味の情報収集
**シーン**: レシピ、旅行先、ガジェットレビューなどの収集
- **Before**: 後で読もうと思って忘れる
- **After**: アーカイブ機能で未読管理、AI要約で内容を素早く把握

### ケース4: 開発者
**シーン**: 技術記事やドキュメントのブックマーク
- **Before**: Qiitaの「ストック」やGitHubのスターが溜まりすぎ
- **After**: 技術タグで分類し、必要な時にすぐ検索

## セキュリティとプライバシー

### データ保護
- Firebase Authenticationによる安全な認証
- Cloud Firestoreのセキュリティルールでユーザーデータを保護
- 他のユーザーのデータには一切アクセス不可

### API キーの管理
- Gemini APIキーは暗号化して保存（準備中）
- ユーザー自身のAPIキーを使用

### 通信の暗号化
- HTTPS通信による安全なデータ送受信
- Firebase SDKによる暗号化通信
